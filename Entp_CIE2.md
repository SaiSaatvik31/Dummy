### 26. Explain Project planning CPM techniques.
[cite_start]The **Critical Path Method (CPM)** is a deterministic project planning technique used when activity times are known and fixed[cite: 1555, 1556].
* [cite_start]**Process:** It involves identifying activities, drawing a network diagram, estimating fixed durations, and calculating earliest/latest start and finish times[cite: 1557, 1561].
* [cite_start]**Critical Path:** It identifies the longest path of dependent tasks, which determines the shortest possible project duration[cite: 1562].
* [cite_start]**Focus:** It focuses on time-cost trade-offs and is widely used in construction and manufacturing where tasks are predictable[cite: 1566, 1794].
* [cite_start]**Slack/Float:** It highlights non-critical tasks that have flexibility (float) without delaying the project[cite: 1567].

### 27. Explain PERT Techniques in detail.
[cite_start]The **Program Evaluation and Review Technique (PERT)** is a probabilistic method used when activity durations are uncertain, often in R&D or new product development[cite: 1569, 1570].
* **Three Time Estimates:** Instead of one fixed time, PERT uses three:
    1.  [cite_start]**Optimistic time ($O$):** Minimum time if everything goes perfectly[cite: 1572].
    2.  [cite_start]**Most likely time ($M$):** The normal time required[cite: 1573].
    3.  [cite_start]**Pessimistic time ($P$):** Maximum time if things go wrong[cite: 1574].
* **Expected Time Formula:** The weighted average is calculated as:
    [cite_start]$$TE = \frac{O + 4M + P}{6}$$[cite: 1576].
* **Variance Calculation:** Risk is assessed using variance:
    [cite_start]$$Variance = \left(\frac{P - O}{6}\right)^2$$[cite: 1782].
* [cite_start]**Usage:** It helps in risk analysis and forecasting completion time in projects with high uncertainty[cite: 1579, 1788].

### 34. Describe in detail about project planning and control using CPM and PERT techniques.
[cite_start]CPM and PERT are essential tools for planning, scheduling, and controlling projects[cite: 1744].
* **Critical Path Method (CPM):**
    * [cite_start]**Nature:** Deterministic (used when times are known)[cite: 1748].
    * [cite_start]**Method:** Decomposes the project into a Work Breakdown Structure (WBS), identifies dependencies, and calculates the critical path (longest path)[cite: 1753, 1750].
    * **Control:** Focuses on critical tasks; any delay here delays the project. [cite_start]It uses float/slack to manage non-critical resources[cite: 1564, 1567].
* **Program Evaluation and Review Technique (PERT):**
    * [cite_start]**Nature:** Probabilistic (used when times are uncertain)[cite: 1770].
    * [cite_start]**Method:** Uses three time estimates (Optimistic, Most Likely, Pessimistic) to calculate an Expected Time ($T_E$) and Variance[cite: 1771, 1782].
    * [cite_start]**Control:** Helps in assessing the probability of meeting deadlines and managing risks in R&D or innovative projects[cite: 1786, 1790].
* **Application:** Both use network diagrams to visualize workflows. [cite_start]They assist in planning timelines, scheduling resources, and monitoring progress to mitigate delays[cite: 1799, 1802].
